# 水晶ERP数据库连接故障诊断指南

## 🚨 问题现象
- 登录API返回：`{"error":"登录失败，请稍后重试"}`
- 服务器健康检查正常，但数据库相关功能失效

## 🔍 系统性诊断步骤

### 1. 检查MySQL服务状态

```bash
# 检查MySQL服务是否运行
systemctl status mysql
# 或者
systemctl status mysqld

# 检查MySQL进程
ps aux | grep mysql

# 检查MySQL端口监听
netstat -tlnp | grep 3307
# 或者
ss -tlnp | grep 3307
```

### 2. 验证MySQL连接

```bash
# 尝试连接MySQL（在服务器上）
mysql -h 127.0.0.1 -P 3307 -u erp_user -p
# 输入密码：erp123456

# 如果连接成功，测试数据库访问
USE shuijing_erp;
SHOW TABLES;
```

### 3. 检查用户权限

```sql
-- 以root用户登录MySQL
mysql -h 127.0.0.1 -P 3307 -u root -p

-- 检查erp_user权限
SELECT user, host FROM mysql.user WHERE user = 'erp_user';

-- 检查数据库权限
SHOW GRANTS FOR 'erp_user'@'%';
SHOW GRANTS FOR 'erp_user'@'localhost';

-- 检查数据库是否存在
SHOW DATABASES LIKE 'shuijing_erp';
```

### 4. 检查应用配置

```bash
# 检查.env文件配置
cat /www/wwwroot/shuijing-erp/api/.env

# 确认配置正确性：
# DB_HOST=127.0.0.1
# DB_PORT=3307
# DB_USER=erp_user
# DB_PASSWORD=erp123456
# DB_NAME=shuijing_erp
```

### 5. 查看应用日志

```bash
# 查看Node.js应用日志
cd /www/wwwroot/shuijing-erp/api
npm start

# 或者查看系统日志
journalctl -u your-app-service -f

# 查看MySQL错误日志
tail -f /var/log/mysql/error.log
```

### 6. 测试数据库连接脚本

```bash
# 在项目目录下运行测试脚本
cd /www/wwwroot/shuijing-erp/api
node test-mysql-connection.js
```

## 🛠️ 常见问题及解决方案

### 问题1：MySQL服务未启动
```bash
# 启动MySQL服务
systemctl start mysql
systemctl enable mysql
```

### 问题2：端口3307未监听
```bash
# 检查MySQL配置文件
cat /etc/mysql/mysql.conf.d/mysqld.cnf | grep port
# 确保port = 3307

# 重启MySQL服务
systemctl restart mysql
```

### 问题3：用户权限不足
```sql
-- 重新授权erp_user
GRANT ALL PRIVILEGES ON shuijing_erp.* TO 'erp_user'@'%';
GRANT ALL PRIVILEGES ON shuijing_erp.* TO 'erp_user'@'localhost';
FLUSH PRIVILEGES;
```

### 问题4：数据库不存在
```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS shuijing_erp CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 导入初始数据
USE shuijing_erp;
source /www/wwwroot/shuijing-erp/mysql_init.sql;
```

### 问题5：防火墙阻止连接
```bash
# 检查防火墙状态
systemctl status firewalld

# 开放3307端口（如果需要）
firewall-cmd --permanent --add-port=3307/tcp
firewall-cmd --reload
```

## 🔧 完整修复流程

### 步骤1：基础检查
```bash
# 1. 检查MySQL服务
systemctl status mysql

# 2. 检查端口监听
netstat -tlnp | grep 3307

# 3. 尝试本地连接
mysql -h 127.0.0.1 -P 3307 -u erp_user -perp123456
```

### 步骤2：权限修复
```bash
# 以root用户登录
mysql -h 127.0.0.1 -P 3307 -u root -p
```

```sql
-- 重新创建用户和权限
DROP USER IF EXISTS 'erp_user'@'%';
DROP USER IF EXISTS 'erp_user'@'localhost';

CREATE USER 'erp_user'@'%' IDENTIFIED BY 'erp123456';
CREATE USER 'erp_user'@'localhost' IDENTIFIED BY 'erp123456';

GRANT ALL PRIVILEGES ON shuijing_erp.* TO 'erp_user'@'%';
GRANT ALL PRIVILEGES ON shuijing_erp.* TO 'erp_user'@'localhost';

FLUSH PRIVILEGES;
```

### 步骤3：验证修复
```bash
# 1. 测试数据库连接
mysql -h 127.0.0.1 -P 3307 -u erp_user -perp123456 -e "USE shuijing_erp; SHOW TABLES;"

# 2. 重启Node.js应用
cd /www/wwwroot/shuijing-erp/api
npm start

# 3. 测试登录API
curl -X POST http://127.0.0.1:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```
