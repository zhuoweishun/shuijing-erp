# 水晶ERP系统阿里云部署指南

## 部署概览

本指南将帮助您将水晶ERP系统部署到阿里云服务器上，包括后端API服务和前端应用。

### 系统架构

```
用户浏览器 → Nginx → Node.js API服务 → MySQL数据库
           ↓
        静态文件服务
```

## 第一步：准备服务器环境

### 1.1 确认服务器配置

- ✅ 阿里云服务器（2核2G，40G存储，3M带宽）
- ✅ 宝塔Linux面板（Alibaba Cloud Linux 3）
- ✅ LNMP环境（Nginx + MySQL + PHP + Node.js）
- ✅ PM2进程管理器

### 1.2 确认数据库

- ✅ MySQL数据库：`shuijing_erp`
- ✅ 数据库用户：`erp_user`
- ✅ 数据表已创建（users、products、purchases）

## 第二步：部署后端API服务

### 2.1 上传后端代码

1. 将 `api` 目录上传到服务器的 `/www/wwwroot/shuijing-erp/` 目录
2. 可以使用宝塔面板的文件管理器或FTP工具

### 2.2 安装依赖

在宝塔面板终端中执行：

```bash
cd /www/wwwroot/shuijing-erp/api
npm install --production
```

### 2.3 配置环境变量

编辑 `/www/wwwroot/shuijing-erp/api/.env` 文件：

```env
# 服务器配置
PORT=3001
NODE_ENV=production
FRONTEND_URL=http://139.224.189.1

# JWT配置
JWT_SECRET=shuijing-erp-secret-key-2024-production-change-this

# MySQL数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=erp_user
DB_PASSWORD=你的数据库密码
DB_NAME=shuijing_erp
```

### 2.4 使用PM2管理API服务

在宝塔面板的PM2管理器中：

1. 点击「添加项目」
2. 填写以下信息：
   - 项目名称：`shuijing-erp-api`
   - 启动文件：`/www/wwwroot/shuijing-erp/api/server.js`
   - 项目目录：`/www/wwwroot/shuijing-erp/api`
   - 运行模式：`fork`
3. 点击「提交」启动服务

### 2.5 测试API服务

访问：`http://139.224.189.1:3001/api/health`

应该返回：
```json
{
  "status": "OK",
  "message": "水晶ERP API服务运行正常",
  "timestamp": "2024-01-15T10:30:00.000Z"
}
```

## 第三步：部署前端应用

### 3.1 构建前端应用

在本地开发环境中：

```bash
# 更新环境变量
echo "VITE_API_URL=http://139.224.189.1/api" > .env

# 构建生产版本
npm run build
```

### 3.2 上传前端文件

1. 将 `dist` 目录中的所有文件上传到 `/www/wwwroot/shuijing-erp/frontend/`
2. 确保文件权限正确（755）

### 3.3 配置Nginx

在宝塔面板中创建网站：

1. 点击「网站」→「添加站点」
2. 域名：`139.224.189.1`（或您的域名）
3. 根目录：`/www/wwwroot/shuijing-erp/frontend`
4. 创建网站

### 3.4 配置Nginx反向代理

编辑网站的Nginx配置，添加API代理：

```nginx
server {
    listen 80;
    server_name 139.224.189.1;
    root /www/wwwroot/shuijing-erp/frontend;
    index index.html;

    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态文件缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 3.5 重启Nginx

在宝塔面板中重启Nginx服务。

## 第四步：配置安全组

### 4.1 开放必要端口

在阿里云控制台的安全组中开放：

- 端口 80（HTTP）
- 端口 443（HTTPS，如果使用SSL）
- 端口 3001（API服务，仅内网访问）
- 端口 8888（宝塔面板）

### 4.2 配置防火墙

在宝塔面板的安全设置中：

1. 开放端口：80, 443
2. 禁用不必要的端口
3. 启用SSH密钥登录

## 故障排除

### API服务无法启动

```bash
# 查看PM2日志
pm2 logs shuijing-erp-api

# 检查端口占用
netstat -tlnp | grep 3001

# 检查数据库连接
mysql -u erp_user -p shuijing_erp
```

### 前端无法访问API

1. 检查Nginx配置
2. 确认API服务运行状态
3. 检查防火墙设置
4. 查看浏览器控制台错误

### 数据库连接失败

1. 检查数据库服务状态
2. 验证用户名密码
3. 确认数据库权限
4. 检查网络连接
