# 阿里云 + 宝塔完整配置指南

## 🎯 指南概述

本指南将帮助您从零开始配置阿里云服务器和宝塔面板，确保水晶ERP系统能够正常运行。我们将详细说明每个步骤在哪个后台操作。

## 📋 两个后台的区别

### 🌐 阿里云控制台
- **作用**：管理服务器硬件、网络、安全等基础设施
- **访问地址**：https://ecs.console.aliyun.com/
- **主要功能**：服务器管理、安全组配置、域名解析、监控等

### 🖥️ 宝塔面板
- **作用**：管理服务器软件、网站、数据库等应用层服务
- **访问地址**：http://您的服务器IP:8888
- **主要功能**：网站管理、数据库管理、文件管理、软件安装等

## 第一部分：阿里云控制台配置

### 1. 安全组配置（重要！）

**位置**：阿里云控制台 → 云服务器ECS → 实例 → 更多 → 网络和安全组 → 安全组配置

#### 需要开放的端口：

| 端口 | 协议 | 用途 | 授权对象 |
|------|------|------|----------|
| 22 | TCP | SSH远程连接 | 0.0.0.0/0 |
| 80 | TCP | HTTP网站访问 | 0.0.0.0/0 |
| 443 | TCP | HTTPS网站访问 | 0.0.0.0/0 |
| 3001 | TCP | API服务端口 | 0.0.0.0/0 |
| 3306 | TCP | MySQL数据库 | 127.0.0.1/32 |
| 8888 | TCP | 宝塔面板 | 您的IP/32 |

#### 具体操作步骤：
1. 登录阿里云控制台
2. 点击左侧菜单「云服务器ECS」
3. 找到您的服务器实例，点击实例ID
4. 点击「安全组」标签页
5. 点击安全组ID进入安全组详情
6. 点击「入方向」→「手动添加」
7. 按照上表逐一添加规则

### 2. 服务器基本信息确认

**位置**：阿里云控制台 → 云服务器ECS → 实例

确认以下信息：
- ✅ 公网IP地址：139.224.189.1
- ✅ 操作系统：Alibaba Cloud Linux 3
- ✅ 实例规格：2核2G
- ✅ 系统盘：40GB
- ✅ 带宽：3Mbps

### 3. 域名解析（可选）

**位置**：阿里云控制台 → 域名与网站 → 云解析DNS

如果您有域名，可以配置A记录指向服务器IP：
- 记录类型：A
- 主机记录：@ 或 www
- 记录值：139.224.189.1
- TTL：600

## 第二部分：宝塔面板配置

### 1. 首次登录宝塔面板

**访问地址**：http://139.224.189.1:8888

#### 获取登录信息：
在服务器终端执行：
```bash
bt default
```

会显示：
- 面板地址
- 用户名
- 密码

### 2. 安装LNMP环境

**位置**：宝塔面板 → 软件商店 → 运行环境

#### 推荐版本：
- **Nginx**：1.20+
- **MySQL**：5.7（适合2G内存）
- **PHP**：8.0+
- **phpMyAdmin**：最新版

#### 操作步骤：
1. 登录宝塔面板
2. 点击左侧「软件商店」
3. 选择「运行环境」标签
4. 点击「一键安装」选择LNMP
5. 选择上述推荐版本
6. 点击「一键安装」
7. 等待安装完成（约10-15分钟）

### 3. 安装Node.js环境

**位置**：宝塔面板 → 软件商店 → 搜索

#### 需要安装的软件：
1. **Node.js版本管理器**
   - 搜索「Node.js版本管理器」
   - 点击「安装」
   - 安装完成后，安装Node.js v16.9.0

2. **PM2管理器**
   - 搜索「PM2管理器」
   - 点击「安装」

### 4. 创建网站

**位置**：宝塔面板 → 网站 → 添加站点

#### 配置信息：
- **域名**：139.224.189.1（或您的域名）
- **根目录**：/www/wwwroot/shuijing-erp
- **FTP**：不创建
- **数据库**：不创建（单独创建）
- **PHP版本**：纯静态

### 5. 创建MySQL数据库

**位置**：宝塔面板 → 数据库 → 添加数据库

#### 配置信息：
- **数据库名**：shuijing_erp
- **用户名**：erp_user
- **密码**：设置一个强密码（记住这个密码！）
- **访问权限**：本地服务器
- **字符集**：utf8mb4

### 6. 配置防火墙

**位置**：宝塔面板 → 安全 → 防火墙

#### 需要开放的端口：
- 80 (HTTP)
- 443 (HTTPS)
- 3001 (API服务)
- 8888 (宝塔面板)

## 第三部分：API部署和配置

### 1. 上传API代码

**位置**：宝塔面板 → 文件 → /www/wwwroot/shuijing-erp/

#### 操作步骤：
1. 点击左侧「文件」
2. 进入 `/www/wwwroot/shuijing-erp/` 目录
3. 点击「上传」
4. 将本地的 `api` 文件夹压缩后上传
5. 解压缩到当前目录

### 2. 安装依赖

**位置**：宝塔面板 → 终端

```bash
cd /www/wwwroot/shuijing-erp/api
npm install --omit=dev
```

### 3. 配置环境变量

**位置**：宝塔面板 → 文件 → /www/wwwroot/shuijing-erp/api/.env

编辑 `.env` 文件：
```env
# 服务器配置
PORT=3001
NODE_ENV=production
FRONTEND_URL=http://139.224.189.1

# JWT配置
JWT_SECRET=shuijing-erp-secret-key-2024-production

# MySQL数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=erp_user
DB_PASSWORD=您在步骤5中设置的密码
DB_NAME=shuijing_erp
```

### 4. 初始化数据库

**位置**：宝塔面板 → 数据库 → phpMyAdmin

1. 点击「phpMyAdmin」
2. 选择 `shuijing_erp` 数据库
3. 点击「SQL」标签
4. 复制粘贴 `mysql_init.sql` 文件内容
5. 点击「执行」

### 5. 使用PM2启动API服务

**位置**：宝塔面板 → 软件商店 → PM2管理器 → 设置

#### 配置信息：
- **项目名称**：shuijing-erp-api
- **启动文件**：/www/wwwroot/shuijing-erp/api/server.js
- **项目目录**：/www/wwwroot/shuijing-erp/api
- **运行模式**：fork
- **Node版本**：v16.9.0

点击「提交」启动服务。

### 6. 配置Nginx反向代理

**位置**：宝塔面板 → 网站 → 您的网站 → 设置 → 配置文件

在 `server` 块中添加：
```nginx
# API代理
location /api {
    proxy_pass http://localhost:3001;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

# 前端路由支持
location / {
    try_files $uri $uri/ /index.html;
}
```

保存并重载Nginx。

## 第四部分：常见问题排查

### 1. API无法访问

#### 检查清单：
- [ ] 安全组是否开放3001端口
- [ ] 宝塔防火墙是否开放3001端口
- [ ] PM2服务是否正常运行
- [ ] 数据库连接是否正常
- [ ] 环境变量是否配置正确

#### 排查命令：
```bash
# 检查端口是否监听
netstat -tlnp | grep 3001

# 检查PM2状态
pm2 status

# 查看API日志
pm2 logs shuijing-erp-api

# 测试API健康检查
curl http://localhost:3001/api/health
```

### 2. 数据库连接失败

#### 可能原因：
- 数据库密码错误
- 数据库用户权限不足
- MySQL服务未启动

#### 解决方法：
```bash
# 检查MySQL状态
systemctl status mysqld

# 重启MySQL
systemctl restart mysqld

# 测试数据库连接
mysql -u erp_user -p shuijing_erp
```

### 3. 前端无法访问API

#### 检查清单：
- [ ] Nginx配置是否正确
- [ ] 前端环境变量是否指向正确的API地址
- [ ] CORS配置是否正确

#### 测试方法：
```bash
# 测试API直接访问
curl http://139.224.189.1:3001/api/health

# 测试通过Nginx代理访问
curl http://139.224.189.1/api/health
```

### 4. 权限问题

#### 文件权限设置：
```bash
# 设置API目录权限
chown -R www:www /www/wwwroot/shuijing-erp/
chmod -R 755 /www/wwwroot/shuijing-erp/
```
